# RootCauseAnalysisSkill

## Description

Systematically investigates problems to identify their fundamental causes rather than treating symptoms.

## Purpose

To provide a structured approach to problem investigation that ensures issues are resolved at their source, preventing recurrence.

## Methodology

1. **Problem Definition**
   - Clearly define the problem or symptom
   - Establish when and where the problem occurs
   - Quantify the problem's impact

2. **Data Collection**
   - Gather relevant data and evidence
   - Document when and how the problem manifests
   - Identify patterns and correlations

3. **Cause Identification**
   - Use structured techniques (5 Whys, Fishbone, Fault Tree)
   - Brainstorm potential causes
   - Distinguish between symptoms and root causes

4. **Cause Validation**
   - Test hypotheses about root causes
   - Verify that identified causes actually produce the problem
   - Confirm that addressing the cause eliminates the problem

5. **Solution Development**
   - Develop solutions that address root causes
   - Implement preventive measures
   - Establish monitoring to prevent recurrence

## Output Guidelines

- **Problem Statement**: Clear description of the issue
- **Root Cause(s)**: Fundamental causes identified
- **Evidence**: Data supporting cause identification
- **Solution Recommendations**: Actions to address root causes
- **Prevention Measures**: Steps to prevent recurrence
- **Monitoring Plan**: How to verify effectiveness

## Example Usage

**Task**: Investigate why a software application has intermittent performance issues.

**Application**:
1. Define: application slows down during peak hours, affecting 30% of users
2. Collect: performance metrics, database logs, user reports, deployment history
3. Identify: database query optimization, memory leaks, network latency, code changes
4. Validate: test each hypothesis, confirm memory leak correlates with slowdowns
5. Solve: fix memory leak, implement memory monitoring, add performance alerts

## Best Practices

- Focus on "why" rather than "what"
- Look beyond obvious causes to deeper systemic issues
- Use multiple analytical techniques for cross-validation
- Involve subject matter experts in cause identification
- Document the analysis process for future reference
- Verify that solutions address root causes, not just symptoms
