# RecommendationSkill

## Description

Formulates clear, actionable, and well-justified recommendations based on analysis, research, or assessment findings.

## Purpose

To transform insights and findings into practical guidance that stakeholders can implement to achieve desired outcomes.

## Methodology

1. **Finding Synthesis**
   - Review all relevant analysis and research findings
   - Identify key insights and conclusions
   - Determine what actions are supported by evidence

2. **Option Generation**
   - Brainstorm potential courses of action
   - Consider multiple approaches and alternatives
   - Evaluate pros and cons of each option

3. **Recommendation Formulation**
   - Select the best course of action based on evidence
   - Ensure recommendations are specific and actionable
   - Align recommendations with objectives and constraints

4. **Justification Development**
   - Provide clear rationale for each recommendation
   - Link recommendations to supporting evidence
   - Address potential objections or concerns

5. **Implementation Planning**
   - Outline implementation steps
   - Identify required resources and timelines
   - Define success metrics

## Output Guidelines

- **Primary Recommendations**: Top-priority actionable items
- **Supporting Rationale**: Evidence and reasoning for each recommendation
- **Implementation Steps**: Clear action items with timelines
- **Resource Requirements**: What is needed to implement recommendations
- **Success Metrics**: How to measure recommendation effectiveness
- **Risk Considerations**: Potential challenges and mitigation strategies

## Example Usage

**Task**: Recommend improvements to a product development process.

**Application**:
1. Synthesize findings: bottlenecks in testing, unclear requirements, poor communication
2. Generate options: agile methodology, automated testing, improved documentation
3. Formulate recommendations: adopt Scrum, implement CI/CD, create requirement templates
4. Justify: industry best practices, case studies, expected efficiency gains
5. Plan: phased rollout, training schedule, success metrics (time-to-market, defect rate)

## Best Practices

- Make recommendations specific and measurable
- Prioritize recommendations by impact and feasibility
- Consider organizational context and constraints
- Provide clear next steps
- Balance short-term wins with long-term improvements
- Include contingency plans for high-risk recommendations
